Analysis of field trip data: density estimation, vegetation sampling

We will spend today analyzing the data we collected in the field last week.

We collected data on density of rocks, using 1 m fixed width transects and using distance sampling truncated to 100 m. We can also treat the data as a 100 m fixed width transect. We will calculate density estimates for each, and compare them.

We collected vegetation data using relevé's, and using line intercept sampling. We will quantify the shrub cover and patch sizes that we estimated with the relevé's and see how they compare.

Density estimates

Transect lengths

For all of the density estimates we will need to know how long our transect was. We will get this by plotting our start and end points (which we recorded in the field using a GPS) in ArcMap, and then we will measure the distance along the trail we walked.

Fixed width transects

Now that you know the lengths of the transects, you can calculate density of rocks of the fixed-width counts.

1. Calculate the areas. The area of the 1 m wide transect is 1 m x transect length in m. Treating the distance sampling counts as a fixed-width transect, the area would be 100 m x transect length in m.

2. Divide number of rocks by transect area. You can get the number of rocks for the 1 m fixed width counts from the "Fixed width transect" database. Divide this number by the transect area to get rocks per square meter.

I will post a class data set for distance sampling, and you can count the number of rocks seen within 100 m of the trail from that. Divide that number by the area of the 100 m transect to get rocks per square meter.

Distance sampling

We will use the R statistical package to get our distance sampling estimates. If you haven't encountered R already, it's an "open source" statistical analysis package. Open source software is typically developed by teams of volunteers, and the "source code" that programmers use to develop the software is made available to anyone who wants to view and modify it. Since the code is available for free, the executable version of open source software is usually distributed free of charge as well. One of the side effects of this development strategy is that users who are also programmers often contribute their own modifications and add-ons for free, in a "share and share alike" manner. The base R package has most of the statistical analysis types you already know about, but contributions from users have vastly increased the types of analyses that R can do.

One such contributed "extension" for R is a package called "unmarked" which includes, among other things, distance sampling functions.

Vegetation sampling

We used the CNPS field forms to collect Releve's of our burned and unburned areas. The Releve's are quick, and semi-quantitative, but we need to check our eyeballometrically recorded cover classes against the estimates we derive from our line transects.

The specific pieces of information we collected that we can check are:

  • Shrub stage (based on percent dead)
  • Percent vascular veg cover
  • Percent shrub cover
  • Percent cover by shrub species
  • Stand size

The stand size we have already checked with ArcMap.

Percent overall shrub cover, vascular veg cover, and cover by shrub species will all come from the line intercept data. The percent living and dead will also come the line intercept data, but in a separate summary table that groups the data by living or dead.

1. Download the class Excel worksheet. I will make a dataset that uses consistent species names, and will post it on the class web site.

2. Calculate the amount of the transects covered by each species. In the worksheet, you will have data on the amount of the tape covered by each shrub (which will just be the end measurement minus the start measurement). You can make a pivot table that sums these amounts by transect and species (use transect ID as row names, species as column names, and use the amount covered as the data field).

Once you have the table, set up a table below with the same row anc column headings. In the body of the table, divide the sums in your pivot table by the length of the transects. This will give you a proportion covered by species for each transect. Some will equal zero, which is fine.

3. Calculate the total of vascular plants and of shrubs. In the next column to the right of your proportions (which you just calculated for 2) enter the column name "Vascular", and sum the proportions of all the vascular plants (which I believe will be everything we sampled). In the next column over, enter the column name "Shrubs", and sum the shrub species.

4. Calculate means and confidence intervals for burned and unburned. Two rows below the bottom of the table of proportions, label column A "Burned". Skip a row, and in the next five rows write "Mean", "s", "n", "Lower", and "Upper".

Next to the Mean label in column A, calculate the average of the burned transects for buckwheat (which should be in column B). In the next row down, next to the s label, calculate the standard deviation for burned transects with the =stdev() function. In the next row, calculate the number of burned transects with the =count() function.

You can calculate the lower limit as mean - t x s / sqrt(n). You can get the t-value with =tinv(0.05, n-1), but replace n-1 with the number of transects minus 1. The upper limit is the same formula, but change the subtraction to addition.

5. Compare the means and confidence intervals to our Releve values. Ideally, the mean proportions should be inside of the cover classes we recorded, but if not then at least the 95% confidence intervals should overlap with the cover classes.

6. Make a pivot table of living/dead shrubs. Make a new pivot table of the transect data with the same layout as before, but add living/dead as column labels above species. You can turn off species that are not shrubs to make the calculation easier.

Add subtotals for living/dead.

Then, in the column to the right of the table, calculate the amount of dead divided by the sum of living and dead. This will be the proportion dead in each transect. Then, you can calculate the mean, s, n, and 95% confidence interval for this proportion dead.
