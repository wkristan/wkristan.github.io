Two-tailed t-tests can detect any difference from μ, so a x̄ that is either above or below μ could be statistically significant.

One-tailed t-tests can only detect differences in one direction - when one tail is selected you are asked whether the alternative hypothesis is "Greater than μ" or "Less than μ" because only differences in the direction you select can be detected.

We are reporting the test results as a p-value - the test is statistically significant if p is less than the alpha level you've selected, 0.05.

With a one-tailed test the p-value is calculated from the observed t-value to the tail indicated as the alternative - with 'Greater than μ' selected the curve is shaded from the observed t-value to positive infinity.

We are reporting the test results using a rejection and retention region. With a two tailed test selected 2.5% of the curve is placed in each of the tails, and we reject the null if the observed t-value falls into this rejection region.

Size of difference that is detectable

To begin there is no difference to detect - the sample mean and population mean are the same.

Set the display to p-value, and increase the value of x̄ to 98.9 - you'll see that x̄ - μ is now 0.3, which increases the observed t-value to 0.237. The p-value for an observed t-value is 0.8179.

Now that you see where the p-value comes from, switch the display to reject/retain regions. Increase the sample mean temperature to 100, and the t-value becomes 1.107 - a difference in temprature of 1.4 degrees is 1.107 standard errors, which has a two-tailed p-value of 0.2971. This is not yet enough to be statistically significant.

Continue to increase the sample mean, and you'll see that when it reaches 101.5 the observed t-value increases to 2.293, which has a p-value of 0.0476 - this is finally statistically significant.

Effect of sample size

Set the sample mean to 98.9, set the Tails to Two, and show the results as Reject/retain region. You'll see that the observed t-value is 0.237, which is solidly inside of the retention region - with this sample size a difference of 0.3 degrees could easily be random sampling variation.

Now start increasing the sample size and watch the observed t-value - you'll see that t gets bigger as the sample size increases. Observed t-value gets bigger with increasing n because the standard error gets smaller - if you look at the observed t-value calculation the standard error gets smaller as n increases, which causes observed t to increase.

Continue to increase the sample size until the observed t-value enters the rejection region in the upper tail - you'll see that when n reaches 686 the difference of 0.3 degrees becomes statstically significant. Increasing sample size increases the precision of x̄ as an estimate of μ, such that a small difference between x̄ and μ can be detected.

Degrees of freedom also go up as n increases, narrows the t-distribution - smaller t-values are needed to fall into the rejection region if the t-distribution is narrow.

Increasing n always increases the chances of detecting a real difference for a two-tailed test, but only does so for a one-tailed test if the difference is in the direction being tested for. Switch to a one-tailed test and set the alternative to less than μ, and you'll see that increasing sample size makes the observed t move away from the rejection region.

One-tailed t-tests

Select the one-tailed t-test option now.

Show the results using the "Reject/retain regions" option, and with the alternative set to Greater than μ you'll see that only an increase in temperature can fall into a rejection region with this choice of one-tailed test.

If you change the alternative to Less than μ the rejection region is placed in the lower tail, and only a decrease in temperature could ever be statistically significant with that choice of one-tailed test.

Note though that the rejection region in the single tail that has one is greater than with a two-tailed test. All of the alpha level's probability is placed in one tail in a one-tailed test, but it is split between both tails for a two-tailed test.

One-tailed testing thus can detect a smaller change than a two-tailed test can, provided it's in the right direction.

Two-tailed t-tests

Two-tailed tests can detect differences in either direction. It is rare that a significant difference in an unexpected direction would be uninteresting, so two-tailed testing is much more common than one-tailed testing.

The p-value in a two-tailed test is obtained by calculating an observed t-value, calculating its negative, and then finding the area under the t-distribution from the positive observed t-value to positive infinity and from the negative observed t-value to negative infinity. These tails represent the random outcomes that would be as far from μ or further - if our observed t-value is bigger than most t-values that would be produced by random chance the p-value will be small. If most random differences are bigger than our observed difference then the p-value will be large.

Initially the sample mean is set to equal the null hypothesis of 98.6, so the t-value is 0, and all of the curve is shaded, giving us a p-value of 1.

If you switch Show results using to "Reject/retain regions" you'll see that an observed t-value that falls far enough from μ in either direction will end up in a rejection region. When the observed t-value is big enough to fall in a rejection region it will also have a p-value that is less than the alpha level we've selected, and thus viewed from either perspective it will be statsitically significant (how much of the tails is shaded is dictated by α, so the p-value method and reject/retain region method are two sides of the same coin).

Change the alpha level

Alpha is the cutoff value we use to decide if a p-value is significant, and to set the sizes of the rejection regions on the t-distribution.

Set the display to reject/retain regions, and then increase the alpha level - you'll see that the rejection region increases in size.

We're able to detect smaller differences if we set alpha to a larger value, but this comes at a price - the alpha level is also the probability of makeing a Type I, or false positive, error. If the null hypothesis is true 5% of the random samples we draw from a population with a mean of 98.6 will fall into the rejection regions and cause us to reject the null mistakenly when it's true.

So, it is possible to detect smaller differences by increasing alpha, but it also increases our false positive error rate. If we want to detect smaller differences it's better to increase sample size, because doing so doesn't increase our false positive rate.

Change the standard deviation

If you decrease the standard deviation the observed t-value will increase (assuming the sample mean is something other than 98.6) because smaller standard deviations result in smaller standard errors. We have a better ability to detect small differences if the standard deviation is small.

The standard deviation is largely a function of the biological organisms we're working with, and we may not be able to control its size the way we can control other aspects of the study. But, we can minimize s by being careful in our data collection, by using homogeneous subjects, or by accounting for any differences that cause variation in the data we're aware of in our analysis.
